# 🤖 StarvellBot Manager

Система управления StarvellBot через Telegram бота с разделением на независимые службы.

## 📋 Возможности

### Управление службами
- **Orders Monitor** - получение и обработка заказов (WebSocket + резервная проверка)
- **Dumper** - автоматическое понижение цен на основе конкурентов
- **Bumper** - поднятие предложений каждые 5 минут

Каждую службу можно запускать/останавливать/перезапускать независимо через Telegram бота.

### Управление конфигурацией
- Редактирование всех настроек через Telegram
- Сохранение в JSON формате
- Автоматическая генерация config.py для совместимости
- Валидация значений
- Маскировка чувствительных данных (токены, ключи)

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Создание Telegram бота

1. Откройте [@BotFather](https://t.me/BotFather) в Telegram
2. Отправьте команду `/newbot`
3. Следуйте инструкциям для создания бота
4. Скопируйте полученный токен

### 3. Получение вашего Telegram ID

1. Откройте [@userinfobot](https://t.me/userinfobot) в Telegram
2. Нажмите "Start"
3. Скопируйте ваш ID

### 4. Настройка

Откройте `main.py` и замените:

```python
# Токен бота (получить у @BotFather)
BOT_TOKEN = "YOUR_BOT_TOKEN_HERE"  # ← Вставьте ваш токен

# ID администраторов (получить у @userinfobot)
ADMIN_IDS = [123456789]  # ← Вставьте ваш ID
```

### 5. Первый запуск

```bash
python main.py
```

### 6. Настройка через Telegram

1. Откройте вашего бота в Telegram
2. Отправьте `/start`
3. Выберите `/config`
4. Заполните необходимые параметры:
   - Starvell API (cookie_session, my_id)
   - Fragment API (hash, cookie)
   - TON Wallet (api_key, mnemonic, destination_address)
   - Сообщения (тексты для клиентов)
   - Производительность (минимальный курс и т.д.)
5. Нажмите "💾 Сохранить и применить"

### 7. Запуск служб

1. Выберите `/services`
2. Нажмите на нужную службу
3. Нажмите "▶️ Запустить"

## 📱 Команды бота

- `/start` - начать работу
- `/services` - управление службами (запуск/остановка/перезапуск)
- `/config` - редактирование конфигурации
- `/status` - показать статус всех служб
- `/help` - справка по командам

## ⚙️ Службы

### Orders Monitor
Отвечает за:
- WebSocket подключение к Starvell
- Получение уведомлений о новых заказах
- Обработку заказов (запрос username, выдача stars)
- Резервную проверку пропущенных заказов

**Запуск**: Orders Monitor запускает обе задачи параллельно (WebSocket + резервная проверка)

### Dumper
Отвечает за:
- Мониторинг цен конкурентов
- Автоматическое понижение цен для топовых позиций
- Соблюдение минимального курса

**Частота проверки**: каждые 30 секунд

### Bumper
Отвечает за:
- Поднятие всех активных предложений
- Поддержание предложений на вершине списка

**Частота**: каждые 5 минут (300 секунд)

## 🔐 Безопасность

- Чувствительные данные (токены, ключи) маскируются в интерфейсе
- Доступ к боту только для указанных ADMIN_IDS
- Конфигурация хранится в отдельном JSON файле
- Автоматическая валидация вводимых значений

## 📊 Статус служб

Для каждой службы отображается:
- Текущий статус (запущена/остановлена)
- Время работы (uptime)
- Последняя ошибка (если была)

## 🛠️ Расширенная настройка

### Добавление новых параметров конфигурации

Отредактируйте `config_manager.py` в методе `_build_schema()`:

```python
'new_section': {
    'name': '🆕 Новый раздел',
    'params': {
        'new_param': {
            'name': 'Новый параметр',
            'description': 'Описание параметра',
            'type': 'string',  # string, int, float, bool, list
            'required': True,
            'default': 'значение по умолчанию'
        }
    }
}
```

### Добавление новых служб

Отредактируйте `service_manager.py` в методе `__init__()`:

```python
'new_service': {
    'name': 'Новая служба',
    'description': 'Описание службы',
    'task': None,
    'running': False,
    'started_at': None,
    'last_error': None,
    'func': self._run_new_service
}
```

И добавьте метод:

```python
async def _run_new_service(self):
    """Запуск новой службы"""
    if not self.bot_instance:
        raise RuntimeError("Bot instance not set")
    
    # Ваш код
    await self.bot_instance.new_service_method()
```

## ❓ Частые вопросы

**Q: Как остановить все службы?**
A: Либо через `/status` → выбрать службу → остановить, либо нажать Ctrl+C в консоли (остановит все автоматически)

**Q: Где хранятся настройки?**
A: В файле `config_data.json`. При сохранении также генерируется `config.py` для совместимости.

**Q: Как сбросить настройки?**
A: Удалите `config_data.json` и перезапустите бота - будут использованы значения по умолчанию.

**Q: Можно ли запускать несколько экземпляров бота?**
A: Нет, рекомендуется запускать только один экземпляр для избежания конфликтов.

## 🐛 Устранение неполадок

### Служба не запускается
1. Проверьте логи в консоли или `bot.log`
2. Убедитесь что все параметры конфигурации заполнены
3. Проверьте интернет-соединение
4. Попробуйте перезапустить службу

### Telegram бот не отвечает
1. Проверьте правильность BOT_TOKEN
2. Убедитесь что ваш ID в списке ADMIN_IDS
3. Проверьте интернет-соединение
4. Перезапустите main.py

### Ошибки при сохранении конфигурации
1. Проверьте права на запись в директорию
2. Убедитесь что значения соответствуют типу (число/строка/булево)
3. Проверьте что обязательные поля заполнены
